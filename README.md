1-кезең: Жобалау және Архитектура (1-ай)
Бұл кезеңде жобаның іргетасы қаланады.


Технологиялық стек: Frontend үшін Next.js, Backend үшін Go тілін таңдау.


Деректер базасы: Мәліметтер базасының архитектурасын жобалау.


Инфрақұрылым: Terraform файлын дайындап, Cloud-Native концепциясы бойынша серверлерді IaC (Infrastructure as Code) ретінде сипаттау.
+1


Жоспарлау: Jira немесе Linear платформасында Өнім бэклогын (Product Backlog) құру.

2-кезең: MVP әзірлеу және Контейнерлеу (2-ай)
Негізгі функционалды құрастыру және автоматтандыру кезеңі.


Scrum процесі: 2 апталық спринттерді бастау және күнделікті 15 минуттық Daily sync кездесулерін өткізу.


Контейнерлеу: Қосымшаны Docker контейнерлеріне салу.


CI/CD конвейері: GitHub/GitLab Actions арқылы автоматты түрде тексеру (Unit-test, Linter) және Docker-бейнесін Registry-ге жіберуді баптау.
+3


Орталарды бөлу: Dev (тестілеу) және Stage (QA үшін) орталарын баптау.

3-кезең: Тестілеу, Қауіпсіздік және Релиз (3-ай)
Өнімді нарыққа шығару алдындағы соңғы дайындық.
+1


Тестілеу: Жүктемелік тестілеу жүргізу және анықталған қателерді түзету.
+1


Қауіпсіздік: Құпия кілттерді сақтау үшін HashiCorp Vault немесе GitHub Secrets қолдану, сондай-ақ Snyk/Trivy арқылы осал тұстарды тексеру.
+1


Мониторинг: Жүйені бақылау үшін Grafana (графиктер) және Sentry (қателерді қадағалау) орнату.


Релиз: 1.0 нұсқасын Prod (клиенттерге арналған орта) ортасына шығару
